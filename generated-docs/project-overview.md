# Project Overview

## プロジェクト概要
- Vim初心者がモーダル編集の概念と操作に成功体験を得るためのジョークアプリです。
- 極端に機能を絞り込み、現在のモードや次に押すべきキーを過剰に表示することで、学習を支援します。
- Rustとratatui/tui-textareaで構築され、Windowsでも動作するターミナルUIエディタとして開発中です。

## 技術スタック
- フロントエンド:
  - `ratatui`: ターミナルUIフレームワークとして画面レイアウトやウィジェット表示を管理します。
  - `tui-textarea`: テキスト編集領域を提供する高機能ウィジェットで、Vimライクな基本操作をサポートします。
  - `crossterm`: クロスプラットフォームでターミナル操作を抽象化し、キー入力やカーソル制御を可能にします。
- 開発ツール:
  - `cargo`: Rustプロジェクトのビルド、依存関係管理、テスト実行などを行う標準ツールです。
- ビルドツール:
  - `cargo`: Rustのビルドシステムであり、プロジェクトのコンパイルと実行を管理します。
- 言語機能:
  - `Rust`: メモリ安全性、並行処理、高性能を特徴とするシステムプログラミング言語で、所有権システムにより安全なコード記述を支援します。

## ファイル詳細説明
- `.gitignore`: Gitのバージョン管理システムが無視すべきファイルやディレクトリを指定します。
- `Cargo.lock`: プロジェクトが依存するクレートの厳密なバージョンとそのハッシュ値を記録し、ビルドの再現性を保証します。
- `Cargo.toml`: Rustプロジェクトのメタデータ（プロジェクト名、バージョン、依存関係など）を定義するマニフェストファイルです。
- `LICENSE`: 本プロジェクトの利用条件を定めるライセンス情報（例: MIT License）を記述しています。
- `README.md`: プロジェクトの概要、セットアップ方法、基本操作、今後の展望などを説明する主要なドキュメントです。
- `_config.yml`: GitHub PagesのJekyllサイト構築に関する設定を定義するファイルです。
- `generated-docs/`: ドキュメント生成ツールによって自動生成された静的ファイルを格納するためのディレクトリです。
- `src/`: プロジェクトのRustソースコードが配置されるディレクトリです。
  - `main.rs`: Rustアプリケーションのエントリーポイント（実行開始地点）であり、主要なアプリケーションロジックが記述されています。

## 関数詳細説明
本プロジェクトはTUIエディタであり、内部的には以下のような役割を持つ関数群で構成されると推測されます。具体的な関数名はコードを参照する必要があります。
- アプリケーション起動関数: プロジェクトのエントリーポイントから呼び出され、ターミナルの初期化、イベントループの開始、UIの描画処理を全体的に調整します。
- イベント処理関数: ユーザーからのキー入力やターミナルイベントを捕捉し、現在のアプリケーションの状態（Vimモード、カーソル位置など）に応じて適切なアクション（テキスト入力、カーソル移動、モード切り替え、コマンド実行など）を実行します。
- UI描画関数: `ratatui`フレームワークを用いて、現在のエディタの状態（テキスト内容、カーソル、アクティブなVimモード、ヒントメッセージなど）に基づき、ターミナル上にユーザーインターフェースをレンダリングします。
- テキスト操作関数: `tui-textarea`ウィジェットと連携し、テキストバッファへの挿入、削除、コピー、移動、単語削除（`dw`）、行削除（`dd`）などの基本的なテキスト編集操作を管理します。
- モード管理関数: Normal、Insert、Command（コマンドラインモード）といったVimモード間の状態遷移を処理し、各モードに応じたキーバインディングとUI表示の切り替えを制御します。
- ファイルI/O関数: エディタのテキスト内容をファイルに書き込む（`:w`）機能や、ファイルから読み込む機能を担当し、永続ストレージとのデータのやり取りを行います。
