# Project Overview

## プロジェクト概要
- Vim初心者がVimの基本的な操作を学び、成功体験を得るためのジョークアプリです。
- 機能を極端に削減し、現在のモードや次に押すべきキーバインドを過剰に派手に表示します。
- Rust製のターミナルUIエディタとして、Vimの主要な操作概念を段階的に学習できるガイドと「卒業試験」を特徴とします。

## 技術スタック
- フロントエンド:
  - `ratatui`: ターミナルUI (TUI) を構築するためのフレームワークです。レイアウト管理やウィジェット描画を効率的に行います。
  - `tui-textarea`: `ratatui`上で動作するテキストエリアウィジェットです。基本的なテキスト編集機能とVimライクなキーバインディングをサポートします。
  - `crossterm`: クロスプラットフォームでターミナルの操作（キー入力、カーソル移動、色制御など）を行うためのライブラリです。

- 開発ツール:

## ファイル詳細説明
- **`.gitignore`**: Gitのバージョン管理から除外するファイルやディレクトリ（例: ビルド生成物、一時ファイル）を指定します。
- **`Cargo.lock`**: `Cargo.toml`で指定された依存関係の具体的なバージョンとハッシュを固定し、再現性のあるビルドを保証します。
- **`Cargo.toml`**: Rustプロジェクトのマニフェストファイルです。プロジェクト名、バージョン、著者情報、依存クレート、機能設定などを定義します。
- **`LICENSE`**: プロジェクトのライセンス情報（著作権や利用条件）を記述したファイルです。
- **`README.md`**: プロジェクトの概要、現状 (WIP)、クイックスタート手順、基本操作、今後の見通しなどが記述された主要な説明ドキュメントです。
- **`_config.yml`**: GitHub Pagesなどの静的サイトジェネレータ（Jekyllなど）の設定ファイルです。このプロジェクトではドキュメントサイトの構築に利用される可能性があります。
- **`generated-docs/`**: プロジェクトのドキュメントが自動生成されて格納されることを想定したディレクトリです。
- **`issue-notes/`**: 開発中の課題、ブレインストーミング、設計に関するメモが格納されるディレクトリです。
  - **`2.md`**: 特定の課題や検討事項に関する詳細なメモです。Vimコマンドの実装に関する議論などが含まれています。
  - **`3.md`**: 別の課題や検討事項に関する詳細なメモです。プロジェクトの機能設計や学習パスに関する議論が記載されています。
- **`src/`**: プロジェクトのソースコードが格納されているディレクトリです。
  - **`main.rs`**: アプリケーションの主要なエントリポイントとなるソースファイルです。TUIエディタのメインロジック、モード管理、イベント処理、UI描画などが実装されています。

## 関数詳細説明
このプロジェクトでは、以下の主要な役割を持つ関数群がアプリケーションの機能を実現していると考えられます。具体的な関数名はコードに依存しますが、ここではその役割を説明します。

- **アプリケーション起動・終了処理**:
  - 役割: アプリケーションの初期設定、ターミナル環境の初期化、メインイベントループの開始、および終了時のリソース解放やターミナル状態の復元を担当する関数群です。
  - 引数: なし、またはアプリケーションの設定オプション。
  - 戻り値: アプリケーションの実行結果を示す値（例: `Result<(), Box<dyn Error>>`）。

- **キーイベント処理**:
  - 役割: `crossterm`を通じてユーザーからのキー入力を非同期的に受け取り、それをアプリケーションが理解できる内部イベント（例: モード変更、カーソル移動、テキスト入力）に変換して処理する関数群です。
  - 引数: `crossterm`のイベントオブジェクト。
  - 戻り値: アプリケーションの内部イベントを表す列挙型など。

- **アプリケーション状態管理**:
  - 役割: Normal, Insert, Visual, CommandといったVimのモード切り替え、カーソルの位置、テキストバッファの内容、Vimコマンドの解析状態など、アプリケーション全体の現在の状態を保持し、イベントに応じて更新するロジックを含む関数群です。
  - 引数: 処理対象のイベント、現在のアプリケーション状態オブジェクト。
  - 戻り値: 更新されたアプリケーション状態オブジェクト。

- **テキスト編集操作**:
  - 役割: カーソルの移動（`h/j/k/l`）、文字の挿入、文字・行・単語の削除（`x`, `dd`, `dw`）、テキストのコピー（`y`）、貼り付け（`p`）など、`tui-textarea`または独自のロジックを用いてテキストバッファに対する具体的な変更を適用する関数群です。
  - 引数: テキストバッファ、カーソル位置、操作の種類、必要に応じてテキスト内容。
  - 戻り値: 変更が適用されたテキストバッファ。

- **UI描画処理**:
  - 役割: `ratatui`フレームワークとウィジェット（`tui-textarea`、`Paragraph`、`Block`など）を使用して、現在のアプリケーション状態（テキスト内容、アクティブなVimモード、強調表示されたキーバインド、カーソル付近のバルーン表示、コマンドライン入力欄など）をターミナルに描画する関数群です。
  - 引数: `ratatui::Frame`オブジェクト、現在のアプリケーション状態。
  - 戻り値: なし（ターミナルへの描画が副作用）。

- **コマンドライン処理**:
  - 役割: Normalモードで`:`キーが押された際にCommandモードに移行し、ユーザーが入力したコマンド文字列（例: `:w`, `:q`, `:wq`, `:q!`)を解析し、ファイル保存やアプリケーション終了といった対応するアクションを実行する関数群です。
  - 引数: 入力されたコマンド文字列、現在のアプリケーション状態。
  - 戻り値: コマンドの実行結果（成功/失敗）および更新されたアプリケーション状態。
